1. Visão Geral

O projeto é uma aplicação web em PHP com MySQL, destinada ao registro, consulta e gestão administrativa de justificativas de faltas escolares.

A aplicação possui duas áreas principais:

  • Portal público: utilizado por responsáveis para enviar justificativas e consultar protocolos.
  • Área administrativa: utilizada por usuários autenticados para consultar, analisar e gerenciar protocolos, alunos e usuários.

A arquitetura é procedural, com separação por diretórios funcionais e arquivos de configuração compartilhados.

2. Arquitetura do Projeto

A estrutura segue uma divisão simples por responsabilidade:

protocolo_faltas/
|-- admin/                 Área administrativa autenticada
|   |-- includes/          Componentes e funções comuns do admin
|-- assets/                Arquivos estáticos de CSS e JavaScript
|-- config/                Configurações gerais e conexão com banco
|-- public/                Páginas do portal público
|-- uploads/               Arquivos enviados pelos usuários
|-- SQL_Cria_BD_Tabelas.sql Script de criação do banco
|-- README.md              Documentação geral do projeto
|-- .htaccess              Proteções básicas no Apache
|-- documento_tecnico.html Este manual técnico

Camadas lógicas

Camada Responsabilidade Arquivos relacionados

3. Pastas e Arquivos

Raiz do projeto

Arquivo/Pasta Função
.htaccess Define proteções básicas no Apache, como bloqueio de listagem de diretórios e negação de acesso direto a arquivos sensíveis como .sql, .zip, .env e logs.
README.md Documentação geral do sistema, com descrição, funcionalidades, instalação e uso.
SQL_Cria_BD_Tabelas.sql Script responsável por criar o banco protocolo_faltas e suas tabelas principais.
documento_tecnico.html Manual técnico do projeto.

config/

Arquivo Função
config/config.php Arquivo de configuração principal. Define BASE_URL, diretório de uploads, tipos permitidos, tamanho máximo, timezone, cabeçalhos de segurança, parâmetros de sessão, funções CSRF e validação de CPF.
config/db.php Cria a conexão com o banco via PDO. Lê configurações de variáveis de ambiente quando disponíveis e usa valores padrão locais como fallback.
config/seed.sql Script de dados iniciais, com usuários e alunos de exemplo. Usado apenas durante a preparação inicial do banco.

public/

Arquivo Função
public/index.php Página inicial do portal público. Apresenta o sistema e direciona para envio ou consulta de protocolos.
public/novo_protocolo.php Formulário público para envio de justificativa. Realiza validações, cria/atualiza responsável, gera número de protocolo, registra o protocolo e salva anexo quando enviado.
public/confirmacao.php Exibe a confirmação de envio com o número de protocolo gerado e dados resumidos.
public/consultar.php Permite consultar o status de um protocolo pelo número gerado no envio.

admin/

Arquivo Função
admin/login.php Tela de login administrativo. Valida usuário ativo, verifica senha com password_verify e inicia sessão autenticada.
admin/logout.php Finaliza a sessão do usuário e redireciona para a tela de login.
admin/dashboard.php Painel inicial administrativo com indicadores por status e lista de protocolos recentes.
admin/protocolos.php Lista protocolos com filtros por busca, status, turma e data da falta.
admin/protocolo_detalhe.php Exibe detalhes completos de um protocolo, dados do aluno, responsável, anexos e permite alteração de status para perfis autorizados.
admin/anexo.php Rota autenticada para servir anexos armazenados em uploads/, evitando acesso direto aos arquivos enviados.
admin/alunos.php Permite cadastro, edição, listagem, busca e exclusão de alunos para perfis autorizados.
admin/usuarios.php Permite ao administrador cadastrar, editar, ativar/desativar e excluir usuários administrativos.

admin/includes/

Arquivo Função
admin/includes/auth.php Verifica se há usuário autenticado e fornece a função requirePerfil() para controle de acesso por perfil.
admin/includes/header.php Cabeçalho HTML compartilhado pelas páginas administrativas.
admin/includes/sidebar.php Menu lateral administrativo, exibindo links conforme o perfil do usuario.
admin/includes/footer.php Rodapé HTML compartilhado e inclusão dos scripts globais.

assets/

Arquivo Função
assets/css/style.css Folha de estilos global usada pelo portal público e área administrativa.
assets/js/main.js Scripts globais para upload drag-and-drop, máscaras de CPF/telefone, menu mobile, alertas temporários, confirmação de exclusão e tooltips.

uploads/

Arquivo/Pasta Função
uploads/ Armazena arquivos anexados aos protocolos. O acesso direto deve ser bloqueado; a abertura deve ocorrer por admin/anexo.php.
uploads/.htaccess Bloqueia listagem, execução de scripts e acesso direto aos arquivos enviados.

4. Fluxos Principais

Envio de justificativa

  1. Responsável acessa public/novo_protocolo.php.
  2. Sistema lista alunos ativos.
  3. Responsável preenche dados da falta, justificativa, CPF, contato e anexo opcional.
  4. Sistema valida CSRF, data, CPF, e-mail e arquivo enviado.
  5. Sistema cria ou atualiza o responsável, registra o protocolo e salva o anexo.
  6. Usuário é redirecionado para public/confirmacao.php.

Consulta pública

  1. Responsável acessa public/consultar.php.
  2. Informa o número do protocolo.
  3. Sistema localiza o registro e exibe status, aluno, data, motivo e observações quando houver.

Análise administrativa

  1. Usuário acessa admin/login.php e autentica.
  2. Acessa admin/protocolos.php para localizar protocolos.
  3. Abre admin/protocolo_detalhe.php para consultar dados e anexos.
  4. Perfis autorizados alteram status e observações do protocolo.

5. Aspectos de Segurança

  • Consultas e alterações no banco utilizam PDO com prepared statements.
  • Senhas de usuários administrativos são validadas com password_verify().
  • Formulários POST usam token CSRF.
  • Saídas HTML relevantes usam htmlspecialchars().
  • Uploads aceitam apenas PDF, JPEG e PNG, com limite de 5 MB.
  • Anexos são servidos por rota autenticada.
  • A pasta uploads/ bloqueia acesso direto e execução de scripts via .htaccess.
  • Arquivos sensíveis na raiz, como SQL e ZIP, são bloqueados por .htaccess.
Recomendação: em ambiente de produção, manter backups fora do diretório público do Apache e configurar credenciais do banco por variáveis de ambiente.

6. Banco de Dados

O banco principal é protocolo_faltas. As tabelas principais são:

Tabela Finalidade
usuario Usuários administrativos, perfis, status ativo e senha criptografada.
aluno Cadastro de alunos, matrícula, turma e contatos.
responsavel Dados do responsável vinculado ao protocolo.
protocolo Registro da justificativa, status, data da falta, observações e vínculos com aluno/responsável/usuário.
anexo Metadados dos arquivos anexados aos protocolos.
